# Avance del artículo
## Sistema web para la gestión de servicios en una lavandería

### Título tentativo
**Diseño e implementación de un sistema web para la gestión de pedidos y seguimiento de servicios en una lavandería urbana**

### Resumen preliminar
En la actualidad, muchas lavanderías pequeñas y medianas continúan gestionando sus pedidos de forma manual, lo cual genera retrasos, errores en el registro de órdenes y dificultades para realizar seguimiento al estado de las prendas. Frente a esta problemática, el presente proyecto propone el diseño e implementación de un sistema web orientado a la automatización básica de procesos operativos en una lavandería.

El sistema desarrollado en este avance permite visualizar servicios, calcular el costo estimado de un pedido según el tipo de atención requerida, registrar órdenes de clientes y consultar su estado mediante un código único generado por el sistema. Para esta primera fase se emplearon tecnologías web base como HTML, CSS y JavaScript, priorizando una arquitectura sencilla, entendible y escalable.

Este trabajo se alinea con principios fundamentales de construcción de software, pues organiza la solución en módulos funcionales, emplea comentarios técnicos para facilitar el mantenimiento y plantea una progresión clara hacia futuras iteraciones con base de datos y autenticación de usuarios.

### Planteamiento del problema
Las lavanderías tradicionales suelen registrar pedidos manualmente en cuadernos, hojas de cálculo o mensajes informales, lo que dificulta la administración del servicio, el cálculo preciso de costos y el seguimiento de cada orden. Esta situación puede afectar la atención al cliente, generar pérdida de información y reducir la eficiencia operativa.

### Objetivo general
Desarrollar un sistema web prototipo que permita gestionar servicios de lavandería mediante módulos de visualización, cotización, registro y seguimiento de pedidos.

### Objetivos específicos
- Diseñar una interfaz web clara e intuitiva para el usuario.
- Implementar un cotizador básico por servicio y peso.
- Registrar pedidos utilizando almacenamiento local.
- Generar códigos de seguimiento para cada orden.
- Presentar una base documental que permita la ampliación futura del sistema.

### Metodología inicial
Para este avance se aplicó una metodología incremental basada en módulos. En primer lugar, se definió el contexto de uso del sistema. Posteriormente, se estableció una estructura de navegación simple y se implementaron funcionalidades mínimas viables que demuestran el objetivo del proyecto.

### Resultados parciales
En esta etapa se logró:
- implementar la interfaz principal del sistema;
- desarrollar el catálogo de servicios;
- construir un cotizador funcional;
- registrar pedidos dentro del navegador;
- consultar el estado de pedidos mediante código único;
- documentar el avance técnico del proyecto.

### Conclusión preliminar
El avance desarrollado demuestra la viabilidad de construir una solución digital básica para una lavandería, aportando organización, trazabilidad y mejor experiencia de usuario. Aunque se trata de un prototipo inicial, sienta las bases para una futura versión con persistencia en servidor, autenticación y reportes administrativos.
